WHY CONTROL FLOW MATTERS
THE BRAIN OF
EVERY PROGRAM
Without control flow, code runs top to bottom — once — and stops. That's not a robot. That's a rock.
🤖
if sensor detects wall → turn left
🌡️
while temp > 80°C → run fan
💡
for each LED → blink 3 times
This is literally how PLCs and microcontrollers work.

IF / ELSE STATEMENTS
MAKE YOUR
CODE DECIDE
int temp = 85; if (temp > 80) { printf("Fan ON\n"); } else if (temp > 60) { printf("Warning\n"); } else { printf("Normal\n"); }
if ( condition ) else if else == != > < >= <=
⚡ In embedded: this is your sensor decision logic. Every robot, thermostat, and PLC uses exactly this.

WHILE LOOP
LOOP UNTIL
CONDITION FAILS
// Run motor while button pressed int button = 1; while (button == 1) { printf("Motor running...\n"); button = readButton(); } printf("Motor stopped.\n"); // do-while: runs AT LEAST once do { printf("Enter valid input\n"); scanf("%d", &input); } while (input < 0);
⚡ The main loop of every Arduino sketch is literally a while(1) running forever.

SWITCH STATEMENT
CLEAN MULTI-
WAY DECISIONS
int mode = 2; switch (mode) { case 1: printf("Manual mode\n"); break; case 2: printf("Auto mode\n"); break; case 3: printf("Sleep mode\n"); break; default: printf("Unknown\n"); }
⚡ In PLCs this maps directly to state machine logic — the backbone of industrial automation.
